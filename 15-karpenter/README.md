# Week 15 — Karpenter

Karpenter is the node autoscaler built for cloud-native workloads — it provisions exactly the right instance type for each pending pod, directly via the EC2 API, without pre-defined node groups. Karpenter questions appear at senior level and signal that the role involves real production cost and scaling work. If you have deployed Karpenter with spot instance diversification, SQS-based interruption handling, and consolidation enabled, you have the specific answers that set you apart.

**Appears in:** senior-level rounds for platform engineer, DevOps, and SRE roles where the job description mentions Karpenter, EKS, node scaling, or cost optimisation.
