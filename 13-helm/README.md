# Week 13 — Helm

Helm is the package manager for Kubernetes — it templates manifests, manages releases, and handles per-environment configuration through values files. Interviews test whether you understand how a generic chart and environment-specific values files separate the what from the where. If you have built a Helm chart used by multiple services across dev and prod, you know the pattern that interviewers are asking about.

**Appears in:** mid-to-senior rounds for any role that mentions Kubernetes, GitOps, or platform engineering. Helm questions are often paired with ArgoCD questions — how does your Helm chart get deployed, and who triggers it?
