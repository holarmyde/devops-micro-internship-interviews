# Week 07 — Ansible (Configuration Management)

Ansible questions test whether you understand idempotent configuration management — the idea that running the same playbook twice produces the same result. Interviewers want to know you can structure a playbook that a team can maintain, not just a script that runs once. Roles, modules, and the difference between `command` and `shell` vs purpose-built modules come up regularly.

**Appears in:** mid-level rounds for infrastructure engineer and DevOps roles that mention configuration management, server provisioning, or on-premise infrastructure.
